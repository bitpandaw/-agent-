# ARCHITECTURE

## 1. Scope & Goal
- 今日目标：从单文件可运行原型，拆分为可扩展工程骨架，并保证功能不回退。
- 当前能力：支持 `calculator`、`search_knowledge`、`query_fault_history` 三个工具。
- 非目标：今天不做性能优化、不做复杂调度策略、不做生产级监控平台。

## 2. Module Contracts

| Module        | 职责                        | 输入                                     | 输出                          | 依赖                                      | 失败处理                                       | 当前代码映射                                            |
| ------------- | --------------------------- | ---------------------------------------- | ----------------------------- | ----------------------------------------- | ---------------------------------------------- | ------------------------------------------------------- |
| entry         | 负责主循环与流程编排        | 用户输入、会话历史、配置                 | assistant 回复、本轮执行状态  | OpenAI client、Chroma collection、config  | 初始化失败直接终止；运行失败进入重试或错误返回 | `rag_system.py`                                         |
| planner       | 决定是否调用工具及工具参数  | `TurnInput`                              | `PlanAction[]` 或直接文本回复 | LLM、`TOOLS_LIST`                         | LLM调用失败重试3次，最终抛出错误               | `rag_system.py` 中 `chat.completions.create` 相关逻辑   |
| executor      | 执行 planner 给出的工具动作 | `PlanAction[]`、运行时依赖（collection） | `ToolResult[]`                | `TOOL_REGISTRY`、参数注入逻辑             | 未知工具/参数错误统一错误语义                  | `rag_system.py` 中 `call_tool` + tool_calls循环         |
| tools         | 提供独立业务能力            | 工具参数                                 | 工具执行结果                  | sqlite3、sentence-transformers、chromadb  | 工具内部捕获异常并返回可读错误                 | `tools/tool_registry.py`                                |
| rag           | 文档分块、索引、向量检索    | 知识文本、query、RAG配置                 | 检索结果或空结果              | embedding model、collection、`config.rag` | 检索失败返回错误；无命中返回空结果             | `rag_system.py` + `tools/tool_registry.py`              |
| state/logging | 记录回放与排错信息          | 用户输入、模型输出、工具事件、错误事件   | 可回放状态记录                | 内存会话对象、日志输出                    | 记录失败不阻塞主流程                           | 当前主要在 `rag_system.py` 的 `conversation` 与 `print` |

## 3. Data Contracts

### 3.1 TurnInput
- `session_id`: string
- `turn_id`: int
- `user_input`: string
- `history`: list

### 3.2 PlanAction
- `tool_name`: string
- `tool_args`: object

### 3.3 ToolResult (统一目标结构)
- `ok`: bool
- `code`: string
- `message`: string
- `latency_ms`: number
- `payload`: object

### 3.4 TurnState
- `session_id`: string
- `turn_id`: int
- `user_input`: string
- `assistant_output`: string
- `tool_events`: list
- `error`: object or null
- `timestamp`: string

## 4. End-to-End Flow
1. entry 读取配置并初始化 client 与 collection。
2. entry 加载知识库文本并执行索引。
3. entry 接收用户输入并组装 `TurnInput`。
4. planner 调用 LLM，生成回复或 `tool_calls`。
5. executor 解析 `tool_calls`，按 `TOOL_REGISTRY` 执行。
6. tools/rag 返回结果，写回会话上下文。
7. planner（LLM）基于工具结果生成最终回复。
8. entry 输出回复并写入 state/logging。

## 5. Failure Paths

| 场景             | 触发点                        | code               | 用户可见信息                              | 重试策略               | state记录                                     |
| ---------------- | ----------------------------- | ------------------ | ----------------------------------------- | ---------------------- | --------------------------------------------- |
| 配置加载失败     | `read_config()`               | `E_CONFIG_LOAD`    | 配置读取失败，请检查 `config/config.yaml` | 不重试，启动终止       | `stage=config, error, timestamp`              |
| LLM调用超时/失败 | planner 调用 chat completions | `E_LLM_TIMEOUT`    | 模型服务暂不可用，请稍后重试              | 最多重试3次            | `stage=planner, retry_count, error`           |
| 工具名不存在     | executor 查 `TOOL_REGISTRY`   | `E_TOOL_NOT_FOUND` | 未知工具：`{tool_name}`                   | 不重试                 | `stage=executor, tool_name, error`            |
| 工具参数不匹配   | `call_tool()` 参数注入        | `E_TOOL_ARG`       | 工具参数错误，请检查输入                  | 不重试                 | `stage=executor, tool_name, tool_args, error` |
| RAG检索异常      | `collection.query()` 抛错     | `E_RAG_QUERY`      | 知识库检索失败                            | 最多重试3次            | `stage=rag, query, error`                     |
| RAG无命中        | 检索结果为空                  | `E_RAG_EMPTY`      | 未检索到相关知识                          | 不重试，返回空结果分支 | `stage=rag, query, top_k, score_threshold`    |
| SQL查询异常      | `query_fault_history` 执行SQL | `E_SQL_QUERY`      | 故障历史查询失败                          | 不重试                 | `stage=tool_sql, sql, error`                  |

## 6. Scalability Rationale (100 tools)
- 工具扩展通过注册表实现，避免主流程堆叠 if-else。
- planner 只产出动作，executor 只执行动作，职责解耦。
- 新增工具主要修改 `tools/tools_json.py` 与 `tools/tool_registry.py`，不破坏主编排。
- 统一 `ToolResult` 让错误处理、日志与测试可标准化。
